Describe the project

This project aims to deploy a prototype for the Web of Things (WoT), specifically solving the major problem of solving the association between IoT devices and Web URLs. We are already halfway considering the features provided of Felooca (exposing services through the Web). The next step is to deploy a web server (or probably make use of Nginx already deployed on the boards) to expose the resources sensors/actuators hosted by the IoT device through the URL made available using Felooca (by adding paths on the URL like /temp, /humidity and so one). Basically, this should be some frontend (html, css, javascript...) + backend (Python, Ruby...) development to have a suitable web interface interacting with the file system of the boards.
